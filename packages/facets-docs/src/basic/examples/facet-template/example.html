Definition

facet-template is a standalone HTML component meant to allow customization of facets inside facet containers, like facet-list, while providing access to the data set to each facet through the use of familiar ES6 template strings syntax:

<facet-template target="facet-term">
	<div slot="header">${metadata.title}</div>
</facet-template>

Setup

A facet-template instance creates a template for a single facet type as defined in its target attribute. Once a target has been defined the content of the facet-template instance acts as the template of for the facet type. At this point the slots defined for each facet are accessible directly from the slot attribute of the facet-template children.

Accessing data

The data defined for each facet is accessible within a facet-template through the use of delimter characters, for example, if the data for a facet is: 

const facetData = {
    ratio: 0.5,
    label: 'a facet!',
    annotation: 'annotation',
    value: 123,
    metadata: {
        title: 'The Title',
        footer: 'footsies',
        left: 'sinister',
        right: 'dexter',
    },
};

The data could be accessed as follows in the facet-template instance: 

<facet-template target="facet-term">
	<div slot="header">${metadata.title} for ${label}</div>
    <div slot="footer">${metadata.footer} of value: ${value}</div>
    <div slot="left">${metadata.left}</div>
    <div slot="right">${metadata.right}</div>
</facet-template>

Which would render as follows:

${metadata.title} for ${label}
${metadata.footer} of value: ${value}
${metadata.left}
${metadata.right}

Tagging

Some times, several different templates are needed for the same facet type, tagging solves that problem by allowing developers to append an arbitrary tag to the facet type. To do so, simply append a # symbol followed by a tag to identify the template:

<facet-template target="facet-term#01">
	<div slot="header">${metadata.title}</div>
</facet-template>

Then add the same tag to the type:

const facetListData = [
    {
        type: 'facet-term#01',
        data: {
            ratio: 0.5,
            label: 'a facet!',
            annotation: 'annotation',
            value: 123,
            metadata: {
                title: 'The Title',
                footer: 'footsies',
                left: 'sinister',
                right: 'dexter',
            },
        },
    },
];

Attributes

facet-template instances also allow developers to customize the attributes of their target facet type. To do so, simply add the attributes that need to be assigned to the facet-template instance itself:

<facet-template target="facet-term#01" actionbuttons="${metadata.actionButtons}">
	<div slot="header">${metadata.title}</div>
</facet-template>

some attriutes can conflict with the facet-template attributes or, depending on the engine used, be sanitized at compile time by templating engines. To solve this issue, it is a good practice to prepend template- to any attribute that needs to be passed down to the facet type itself:

<facet-template target="facet-term#01" template-style:"background-color:${metadata.bg}">
	<div slot="header">${metadata.title}</div>
</facet-template>
